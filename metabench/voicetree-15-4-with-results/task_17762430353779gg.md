BACKGROUND
═══════════════════════════════════════════════════════
The HCH benchmark (/Users/bobbobby/repos/voicetree-evals/metabench/hch/spec_corrected.md) currently runs as a SINGLE `llm.prompt()` call per question. The model emits ATOMIC_PREDICTION + SUBTASKS + SUB work blocks + ANSWER all in one context. The "decomposition" is stylistic markup, not architectural isolation.

The original HCH concept (Factored Self-Assessment / Factored Cognition — Ought, Paul Christiano et al.) envisions TRUE decomposition: a parent spawns independent child sessions, each seeing only its assigned subtask + no shared context, then integrates the children's outputs. This gives:
- Real context isolation per subtask
- Actual per-subtask token accounting
- Architectural (not rhetorical) capability boundary testing
