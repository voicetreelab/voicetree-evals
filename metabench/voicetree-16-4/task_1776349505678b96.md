# Factory-A Validator Pipeline — generate → Gemini Flash → parse → verify → score

You are the **Factory A validator builder**. Mission: build a reusable dev-time CLI that validates a single (class, difficulty, seed) end-to-end by calling a real LLM. This tool IS the load-bearing unit Factory A Codex agents will call at scale to confirm each instance is well-posed before committing.

## Why this exists

Before spawning 6 A-Codex to generate 120+90 instances in parallel, we need to prove:
1. Each class's generator produces a valid instance
2. The instance renders to a coherent NL prompt
3. A real LLM can parse the prompt and emit a parseable submission
4. The verifier runs on that submission and returns `(score, feasible, details)`
5. Gold + baseline are computed cleanly

If this loop breaks for any class, we catch it now — not after 20 wasted Codex-hours.

## Scope

**Port + build, don't redesign.** Reuse existing code wherever possible:
- **Reference setup (READ + PORT):** `hch/portfolio_v2/` — this has the working generate+verify+LLM-call pattern. You are authorized to read this directory.
- **Source baselines (READ + USE DIRECTLY):**
  - `hch/portfolio_spike/cjs_instance.py`, `steiner_coloring_instance.py`, `graph_coloring_instance.py`, `tsp_instance.py`
  - `hch/treewidth_mwis/` — MWIS
  - `hch/bayesnet_ve/` — VE
  - `hch/masked_block_jobshop/protocol.py`, `prompt.py`, `render_nl.py`, `gemini_client.py` — protocol + Gemini client
- **LLM:** `gemini-2.5-flash` (cheap, ~$0.001/call — fine for 210 dry-runs). Use `google-genai` SDK.

## Deliverable

A CLI script:
```
python metabench/kaggle_submission/scripts/verify_pipeline.py --class cjs --difficulty medium --seed 1
```

That:
1. Imports the class's generator from `hch/...`
2. Generates instance at (difficulty, seed)
3. Renders to NL via ported `render_nl.py` logic
4. Calls Gemini 2.5 Flash with system prompt + rendered instance (system prompt per `kaggle-submission-design.md` / `llmpromptflowanswer.md`)
5. Parses submission from raw output (regex, per v2 design — no extractor LLM)
6. Imports the class's verifier, runs `verify(instance, submission) → (score, feasible, details)`
7. Prints: `PASS/FAIL, score=X, feasible=Y, wall=Zs, details=...`
8. Saves raw transcript to `tests/fixtures/{class}_{diff}_seed{seed}.txt`

## Success gate (before reporting back to Ayu)

1. Tool runs end-to-end on **at least 2 classes** (recommend: TSP + CJS — best-documented baselines)
2. Verifier returns `feasible=True` with sensible `score` on Gemini Flash's submission
3. Gold + baseline objectives computed (OR-Tools)
4. Transcripts saved to `tests/fixtures/`
5. CLI has clear `--help` output so A-Codex can invoke it at scale
6. One end-to-end call takes <60s (so 210 calls at scale is ~2-3h serial, minutes parallel)

## Escalate to Ayu/user immediately

- Any class's generator fails or produces infeasible instances with high frequency (>20%)
- Gemini Flash output is un-parseable without an extractor LLM (would break v2 design)
- `hch/portfolio_v2/` setup pattern doesn't port cleanly — need to discuss alternatives
- MWIS or VE verifier contract differs materially from other classes (risk for scale)
