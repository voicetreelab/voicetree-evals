# Unified LLM CLI Setup — gemini-3-lite, sonnet-4.6, gpt-5.4-mini

Mission: get a CLI up and running that lets us invoke three frontier-tier-lite models from the command line with a consistent interface. This becomes the substrate for our multi-model evaluation phase (~45 runs, ~$75 budget).

## Models (exact targets)
- **Gemini 3 Lite** — Google (google-genai SDK)
- **Claude Sonnet 4.6** — Anthropic (anthropic SDK)
- **GPT-5.4 mini** — OpenAI (openai SDK)

These are all 2026 models — may need latest SDK versions. Use the `find-docs` skill or web search to verify exact model IDs as of today (2026-04-17).

## Decision: existing tool vs build

**Research (15-20min max) — pick one path:**

1. **`llm` by Simon Willison** — check if plugin coverage includes all 3 brand-new models. Install + test if yes.
2. **`litellm`** — unified interface, check model-ID support. Install + test if yes.
3. **Custom ~50-line Python wrapper** — if the above have gaps for these very new models, just write it. Use 3 SDKs directly. Prefer this if any uncertainty — it's faster than debugging plugin version mismatches.

**Recommendation:** default to option 3 (custom wrapper) unless `llm` or `litellm` clearly works out-of-the-box for all three model IDs. We don't have time to chase plugin bugs.

## Deliverable

CLI invocation shape (whichever tool you pick):
```
<cli> --model gemini-3-lite --system "..." --prompt "..."
<cli> --model sonnet-4.6 --system "..." --prompt "..."
<cli> --model gpt-5.4-mini --system "..." --prompt "..."
```

Must support:
- System prompt + user prompt (required for our benchmark's prompt structure)
- Temperature / max-tokens flags
- Returns raw text output to stdout
- Logs token usage + cost to stderr (for budget tracking)
- Exit code 0 on success, non-zero on error
