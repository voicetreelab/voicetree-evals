## Learnings

1. Seed fallback belongs in the dataset builder, not in the generators, if the policy is “first successful actual seed wins” and the row id must reflect that actual seed.
2. `mwis` hard is the only class that needed fallback in this run; requested seed 3 deterministically fails the bridge-separator pre-check, but seed 4 succeeds cleanly.
3. Some “hard” rows are materially the same as medium for a given seed because those generators encode hardness through acceptance thresholds rather than a different base size; that is expected but worth flagging for later benchmark curation.
