## Deliverables

### `harness/scoring.py`
Two functions:
- `score_solo(gap_pct: float, feasible: bool, wall_s: float) -> float`
  Returns `max(0.0, 100.0 − gap_pct) − 0.01 * wall_s` when feasible, else `0.0`. Clamp to `[0, 100]` BEFORE time penalty? Read design doc: the design says "floored at 0" for the value component, then time cost subtracts. So: `max(0, 100 - gap_pct) - 0.01 * wall_s` — time cost CAN push below zero. Return as-is (Kaggle accepts negative scores).
- `score_portfolio(components: list[dict], wall_s: float) -> float`
  Per design: `Σ value_cap · headroom − 0.05 · wall_s` where headroom is `max(0, 1 − gap_pct/100)` for each component. Time penalty coefficient doubles (0.05 vs 0.01) per design.

### `harness/runner.py`

Main entry: `run_instance(llm, instance: dict, cls: str, difficulty: str, seed: int, gold_objective: float, baseline_objective: float, value_cap: float, *, components: list | None = None) -> dict`

Returns a row dict with all fields S3 will embed into the `@kbench.task` return:
```
{
  "score": float,                    # final economic score (what @kbench.task returns)
  "score_at_stop": float,            # score if we stopped where the model said stop
  "score_after_cf": float,           # score after the forced +1 turn
  "cf_delta": float,                 # score_after_cf - score_at_stop
  "wall_s": float,
  "n_exec_turns": int,
  "stop_reason": str,                # "decision_stop" | "turn_cap" | "wall_budget"
  "transcript": list[dict],          # [{"role","content"}, ...] — for analyzer
  "parsed": dict,                    # last parsed BEST_GUESS, DECISION, FORECASTS
  "cf_parsed": dict,                 # parsed CF turn (BEST_GUESS, FORECASTS, no DECISION)
  "error": str | None
}
```

Loop shape (pseudocode):
```
1. Build system prompt + Turn 1 user message via harness.prompt.
2. llm.prompt(messages=[sys, turn1]) → response_1. Append to transcript.
3. Parse response_1 via protocol: PLAN_STATE, NEXT_SUB.
4. For turn_n in 2..MAX_EXEC_TURNS:
     - Check wall: if elapsed_s + expected_turn_s > TOTAL_BUDGET_S - CF_RESERVE_S → force stop_reason="wall_budget", break.
     - Build Turn N user message with TIMING BLOCK (elapsed_s, remaining_s, last subtask budgeted/actual, history).
     - llm.prompt(...) → response_n.
     - Parse: SUB_n, BEST_GUESS, UPDATED_PLAN_STATE, QUALITY_FORECAST, CONTINUE_FORECAST, DECISION, NEXT_SUB.
     - If DECISION == "stop" → stop_reason="decision_stop", break.
5. (at stop) score_at_stop = score_solo(...) using verifier(instance, last_BEST_GUESS).
6. CF FORK: build one more Turn N user message with DECISION/NEXT_SUB contract elements STRIPPED (forced continue). Same transcript, +1 turn.
     - Track forced_wall_s separately; cap at CF_RESERVE_S.
     - Parse CF response → new BEST_GUESS.
     - score_after_cf = score_solo using verifier on CF BEST_GUESS, wall_s = total including forced.
7. cf_delta = score_after_cf - score_at_stop.
8. Return final dict. `score` field = score_after_cf (headline — see design "CF is the measured score, stop-rationality is the signal").
```
