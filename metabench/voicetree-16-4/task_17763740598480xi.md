# Metacog Forecast Analysis — score QUALITY_FORECAST + CONTINUE_FORECAST over overnight 206-row set

You are an Opus investigator. The overnight pilot captured per-turn metacog forecasts on every LLM run, but nobody has aggregated or scored them yet. The reviewers (Tao R1, Ayu_1 R2) and the discussion writer (Ben) all focused on strict parse rate + joint feasibility + score/gap. Metacog is dark data. Your job: illuminate it.

## What metacog is in this codebase
Per-turn structured forecasts the model emits in each exec turn:
- `quality_forecast` — `{p_gap_le_2, p_gap_le_5, p_gap_le_10}` for solo classes (portfolio may differ)
- `continue_forecast` — `{p_improve, expected_delta_score, expected_gap_reduction}`
- Sometimes `atomic_forecast` — same shape as quality_forecast but for single-step subs

Emission spec + parser:
- `kaggle_submission/harness/prompt.py:120-133` — what the model is asked to emit
- `kaggle_submission/harness/protocol.py` — runtime parser
- `kaggle_submission/kaggle/notebook.ipynb` — `parse_quality_forecast`, `parse_continue_forecast`

## Where the data lives
- `kaggle_submission/results/runs/{model}/{row_id}.json` — per-row artifacts (used to aggregate across rows)
- `kaggle_submission/results/full/{row_id}/{model}.json` — per-row full transcripts
- Per-turn dicts: look for top-level `turns` array, each turn has `quality_forecast` + `continue_forecast` keys (confirmed in e.g. `results/runs/claude-sonnet-4.6/portfolio_medium_seed1.json` lines 414, 421, 1403, 1410)

## What you should produce
Write a small script (e.g., `kaggle_submission/scripts/analyze_metacog.py`) that:
1. Walks `kaggle_submission/results/runs/{model}/*.json`
2. Extracts per-turn `quality_forecast` + `continue_forecast` and the realized gold score / feasibility from the same record (or from `results/full/{row_id}/question.json` if needed for ground truth)
3. Computes per-model × per-class × per-difficulty:
   - **Quality Brier**: for each turn's `p_gap_le_{2,5,10}` forecast, score against realized gap (`(p - I[gap_le_k])^2`)
   - **Continue calibration**: compare final-turn `p_improve` against whether the next turn actually improved; compare `expected_delta_score` against realized delta
   - **Final-turn vs mid-run forecast drift**: does the model's confidence move monotonically toward the ground truth?
4. Produces a markdown report with:
   - Per-model × per-class Brier table (G / S / GPT × 6 solo classes × 2 difficulties + portfolio)
   - Continue-forecast calibration table (mean `p_improve` vs observed improvement rate)
   - 3-5 headline findings (e.g., "Sonnet is systematically overconfident on MWIS", "GPT continue-forecast p_improve is decorrelated from actual improvement")
   - Notes on missing / malformed forecasts (how often was the field absent)
