## Mission

Get our Metacognition-track benchmark submitted to Kaggle by the 09:59 GMT+10 deadline (2026-04-17). Two phases:

1. **Dry-run NOW** — de-risk Kaggle quirks before we're under deadline pressure.
2. **Real submit LATER** — after Factory A (Ivan) finishes Phase 1+2 and we have real evaluation results.

## Required deliverables (per competition spec)

- Kaggle Benchmark link: `kaggle.com/benchmarks/<user>/<name>` — must exist + load
- Writeup: ≤1500 words, 3 pages, competition-provided template, Metacognition track
- Cover image: mandatory
- Public notebook: optional
- Final submit action before deadline

## First action: work with the user

1. **Scout Kaggle rules** — read the hackathon page, surface:
   - Resubmission policy (can we overwrite after first submit?)
   - Writeup template format + location
   - Per-benchmark evaluation quotas / rate limits
   - Private vs public benchmark timing (spec says "private until deadline, auto-public after")

2. **Propose dry-run plan** — post as progress node + ping Ayu/user for review.
   - Throwaway slug: e.g. `<user>/metacog-test-0` — don't burn the real slug `metacog-optim-v1`
   - Use Ben's smoke artifacts (or 1-row toy) to test: auth → upload → benchmark page renders → `.evaluate()` fires
   - Target: answer 3 load-bearing open gaps:
     (a) Does `.evaluate()` have per-row timeout <1800s?
     (b) Is "Evaluate More Models" UI-only or API-accessible?
     (c) Does `EMBEDDED_MODULES` actually work under Kaggle's Python runtime?

3. **Get user sign-off before** any action that could consume the real slug or publicly visible Kaggle state.
