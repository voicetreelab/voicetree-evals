# Compute M1 (p_solve Brier + Murphy decomposition) and add it to analyze_metacog.py

You are an Opus investigator. Your predecessor (Anna, same task) built
`kaggle_submission/scripts/analyze_metacog.py` which computes M2/M3/M4/continue-forecast
calibration, but left M1 as "NOT COMPUTED" because M1 needs per-turn verifier scoring. Your
job: compute it, wire it into the script, and re-render the report.

## What M1 is (from writeup-v2.md §Scoring decomposition)

> **M1 (subtask solvability Brier):** mean(`p_solve_k − outcome_k`)² across all subtasks
> where outcome is `kept_as_best` after verification.

Where:
- `p_solve_k` = the `p_solve` field in the NEXT_SUB that kicked off subtask `k` (emitted in the
  plan turn for subtask 1, and in each exec turn's NEXT_SUB for subtasks 2+).
- `kept_as_best` = 1 if subtask `k`'s `best_guess` became the running-best (higher verified
  score than all prior subtasks' best_guess), else 0.

Then: Murphy decomposition → `reliability`, `resolution`, `uncertainty` (same approach as
the M2 decomposition already in the script — see `murphy_decomposition(pts)` there).

## Why this matters

Writeup-v2's finding 6 (small-tier models have distinct metacog failure) and finding 7
("resolution reveals what Brier hides") are the only findings that currently cite M1 numbers.
Without M1 we either drop those findings or re-ground them on M2 — but M2 inverts finding 7's
direction (writeup claims GPT has high resolution; our M2 says GPT has near-zero). So M1
is load-bearing for the writeup as written.

## What to do

1. **Parse p_solve per subtask per row.** For each `results/full/{row_id}/{model}.json`:
   - Plan turn (transcript[0]): parse with `harness.protocol.parse_plan_turn` → `next_sub.p_solve`
     is subtask 1's p_solve.
   - Exec turns (transcript[1:] with role=assistant): parse with
     `harness.protocol.parse_exec_turn_partial` → `next_sub.p_solve` is the *next* subtask's
     p_solve (only present when decision=continue).
   - Subtask indexing: subtask 1 from plan, subtask 2 from exec turn 1's NEXT_SUB, etc.

2. **Score each subtask's best_guess with the verifier.**
   - Each exec turn's assistant content contains a `BEST_GUESS:` block — parse with
     `harness.protocol.parse_best_guess`. This is subtask `k`'s submission.
   - Verifier: `from verifiers import CLASS_TO_VERIFIER; verifier = CLASS_TO_VERIFIER[cls]`.
     Signature is `verify(instance, submission) → (score, feasibility, failure_reason, details)`
     — confirm by reading one of `verifiers/{cjs,tsp,ve,steiner,mwis,graphcol}.py`.
   - Instance: `json.loads((row_dir / "question.json").read_text())["instance"]` (or similar —
     check a question.json's shape first).
   - `kept_as_best[k] = 1` if `score_k > max(score_1..score_{k-1})` else 0. Subtask 1 is
     kept_as_best=1 by convention (no prior to beat) IF it's feasible; else 0.

3. **Handle edge cases:**
   - **Portfolio:** BEST_GUESS is a nested dict `{cjs: {...}, tsp: {...}, steiner: {...}}`. The
     portfolio verifier lives at `verifiers/portfolio.py` (or invoked via the same registry).
     Likely needs its own path. If you can't verify portfolio subtasks cleanly in under
     20 minutes, skip portfolio and compute M1 on solo classes only — note it as a caveat.
   - **VE:** uses different gap thresholds but the verifier is standard.
   - **Missing best_guess / parse fail:** skip that subtask (don't count toward M1 denominator).
   - **p_solve out of range / missing:** skip.
   - **Feasibility check:** an infeasible best_guess scores nothing, so `kept_as_best=0`.

4. **Add to analyze_metacog.py:**
   - Extend `model_level_stats(rows)` to compute M1 alongside M2.
   - Emit `m1_brier`, `m1_reliability`, `m1_resolution`, `m1_uncertainty`, `m1_n` per model.
   - Replace the "NOT COMPUTED" placeholder rows in the profile table with real numbers.
   - Re-run the script.

5. **Sanity-check output:**
   - M1 Brier should be in [0, 1]. If > 0.5, something is wrong with outcome labeling.
   - Murphy identity: `brier = reliability − resolution + uncertainty` should hold to 3 decimals.
   - n should be ≥ mean_exec_turns × 64 rows ≈ 80-ish per model minimum (more with multi-turn runs).
