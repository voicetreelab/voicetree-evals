# What to build

## Instance generator (`jobshop_instance.py` rewrite)
25 jobs × 15 machines. Hidden generation (the model must not see any of this structure — it sees only the resulting operation matrix + durations + due dates + tardiness weights):
- **SBM families**: 4 clusters, 6–7 jobs each. Each cluster has a preferred 5-machine set. 80% of a cluster's operations routed through preferred machines; 20% routed through other clusters' preferred machines. Families overlap.
- **Bridge jobs**: 2 per cluster, routed through 2+ families' preferred machines deliberately.
- **Shared machines**: 3 machines used heavily cross-family (cross-family load concentration).
- **Outliers**: 5 jobs at 3× processing time, scattered across clusters.
- **Due dates**: 30% of non-outlier jobs with tight deadlines; weighted tardiness in the objective. Other 70% loose.
- **Objective**: weighted makespan + weighted tardiness. Tune weights so no single axis (bottleneck-first / family-first / outlier-first / due-date-first) dominates — ablate against pure heuristics and confirm spread of outcomes BEFORE doing the LLM run.

## Gold + baseline
- **Gold**: CP-SAT on composite objective. Timeout to a generous bound (e.g. 10 min) — if CP-SAT doesn't prove optimal at 25×15, record best-found and mark.
- **Baseline**: SPT dispatch. Confirm baseline-to-gold gap is meaningful (target ≥ 20%).

## Pre-flight sanity check (MUST do before LLM run)
Run all 4 named decomposition heuristics as pure code on the instance:
1. Bottleneck-first (schedule the top-3 loaded machines first)
2. Family-first (detect via modularity on job-machine bipartite, solve per cluster, reconcile)
3. Outlier-first (sequence 5 long jobs then fill)
4. Due-date-first (EDD ordering)

Confirm they produce a **spread** of final qualities (e.g. 15–35% spread between best and worst heuristic). If they all converge within 5% of each other, the instance is NOT indeterminate — adjust SBM overlap / tardiness weight and regenerate. Log the spread in the progress node.

## Protocol adaptation (`protocol.py`)
Reuse codex_metagame_v2's plan-turn + multi-exec-turn structure (this is the "multiple subtask LLM calls" setup the user referenced — keep it). Add to the plan-turn schema:
- `declared_decomposition_axis`: one of {`bottleneck_first`, `family_first`, `outlier_first`, `due_date_first`, `composite`, `other`}
- `declared_axis_rationale`: free text
- `declared_boundary_cut`: free text describing which jobs/machines the model groups together
Keep per-subtask p_solve declarations as in existing harness.

## Render NL (`render_nl.py`)
Present the instance as operation matrix + durations + due dates + tardiness weights. Do NOT expose family labels, bottleneck markers, or outlier tags. The "mask" is critical.

## Run

- gap_pct
- declared axis + rationale + cut
- number of exec turns, subtask p_solve declarations, realized per-subtask quality
- wall time, stop reason
