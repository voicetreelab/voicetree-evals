
**Generator** (`steiner_coloring_instance.py`): seeded procedural generator producing the same structural class — N villages as 2D points (random), cable graph as k-nearest (say k=3) or distance-threshold graph with Euclidean costs, K terminals sampled uniformly, interference graph with 1-2 planted triangles chosen to ensure the coupling fires. ## 2. Gold solver (`steiner_coloring_gold.py`)
Joint ILP via OR-Tools CP-SAT. Variables:
- edge_used[e] ∈ {0,1} for each cable edge
- freq[v] ∈ {0, ..., K_MAX} for each village (K_MAX = 6 is plenty for N=8)
- active[v] ∈ {0,1} (village is on the network iff incident to a used edge or is a terminal)

Constraints:
- Tree connectivity: each terminal reachable via used edges (flow formulation)
- Coloring: for each interference pair (u,v): if active[u]=1 AND active[v]=1, then freq[u] ≠ freq[v]
- Count distinct frequencies used (via indicator vars num_freq_used[k])

Objective: minimize sum(edge_cost[e] * edge_used[e]) + 15 * sum(num_freq_used[k])

If you can't get CP-SAT to encode this cleanly in 1-2 hours, fall back to a brute-force enumeration for N ≤ 10 (try all edge subsets forming trees over terminals, color greedily, take min). Canonical instance has N=8 — brute force is fine as a backup.

Save gold per instance: `(optimal_cost, tree_edges, freq_assignment)`.

## 3. Verifier (`verify.py`)
