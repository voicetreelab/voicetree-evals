# Treewidth MWIS spike v1 — build local harness, run one Gemini 3 Pro seed

# Goal
Build a LOCAL spike at `hch/treewidth_mwis/` and run it ONCE on ONE Gemini 3 Pro configuration, ONE seed. Do NOT build a Kaggle port. Do NOT run multiple seeds/models. One progress node at completion.

# Source of truth for problem design
Follow `/Users/bobbobby/repos/voicetree-evals/metabench/voicetree-15-4/treewidth-mwis-proposal.md` exactly — that is the frozen spec for instance generation, decomposition axes, gold, and metacog signals. Pre-flight checks listed in the proposal's "Open design questions" are expected; scale the instance to 150-180 nodes if 120 is too easy.

# Harness pattern to REUSE
Copy the protocol shape from `hch/masked_block_jobshop/` — specifically from the planstate variant (see `hch/masked_block_jobshop/prompt.py`, `protocol.py`, `run_spike.py` and the `masked_block_jobshop_gemini3pro_seed1_planstate_20260416.jsonl` result). Key invariants:
- Turn 1 = plan turn only. Emits `DECLARED_DECOMPOSITION_AXIS` (enum: balanced_separator | modularity | high_degree_peel | weight_greedy | composite | other), `DECLARED_AXIS_RATIONALE`, `DECLARED_BOUNDARY_CUT` (JSON list of vertex IDs the model claims is a separator), `PLAN_STATE` (free-form text), `ATOMIC_FORECAST` (thresholded gap%), `CONTINUE_FORECAST`, `DECISION`, `NEXT_SUB`.
- Exec turns receive `CURRENT_PLAN_STATE`, emit `BEST_GUESS` (JSON: `{"selected_vertices": [...], "total_weight": <int>}`), `UPDATED_PLAN_STATE`, `QUALITY_FORECAST`, `CONTINUE_FORECAST`, `DECISION`, optional `NEXT_SUB`.
- `PLAN_STATE` is free-form TEXT (not schema'd). The user explicitly asked for todo/subtask plan state as a string.
- 30-min total budget, 600s per-subtask budget, MAX_EXEC_TURNS=4.
- Thresholded gap: 100*(opt - model_value)/opt. Thresholds: ≤2%, ≤5%, ≤10%.

# Deliverables
- `hch/treewidth_mwis/graph_instance.py` — SBM+decoys generator, CP-SAT MWIS gold, greedy baseline
- `hch/treewidth_mwis/prompt.py`, `protocol.py`, `render_nl.py`, `gemini_client.py`, `run_spike.py` (can be near-copies of masked_block_jobshop versions)
- `hch/treewidth_mwis/results/treewidth_mwis_gemini3pro_seed1_planstate_<YYYYMMDD>.jsonl`
- ONE progress node at `voicetree-15-4/` mirroring the result-row schema from `masked-block-jobshop-v1-gemini3pro-seed1-result.md`. Include: final objective, gap%, declared axis, exec turn count, stop reason, wall seconds, initial/final PLAN_STATE, feasibility of declared cut (did the claimed separator actually disconnect the graph?).

# Model config
- `models/gemini-3-pro-preview`, seed 1.
- GEMINI_API_KEY is already in the env that masked_block_jobshop uses — reuse that config.

# Verification requirements
- Feasibility check on BEST_GUESS: selected set must be independent (no edge between any two selected vertices). Weight claim must match sum of weights. Mirror the masked_block_jobshop verifier style.
- Feasibility check on declared cut: does removing those vertices actually disconnect the graph? Log separator_size and did_separate fields.

# Scope discipline
Do NOT add Kaggle. Do NOT add multi-model sweeps. Do NOT redesign the protocol — if something feels off, document it as an open question in the progress node, don't refactor the contract. Mirror the existing masked_block_jobshop idioms so the analyzer can be reused trivially later.
