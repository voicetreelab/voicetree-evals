# Literature survey: NP-hard search problems where heuristics are instance-specific

# Your role
You are a **researcher**. Your job is a literature survey, not implementation. Do NOT write code. Produce a written synthesis of candidate problem classes for an LLM metacognition benchmark.

# Context — why this question matters
We are designing a Kaggle metacognition benchmark for frontier LLMs. The benchmark idea: give the model an NP-hard problem, a wall-clock and token budget, and score `$score = ACCURACY_REWARD · (100 − gap_pct) − TIME_PENALTY · wall_seconds`. The model must decide when to stop, how to decompose, and self-assess — the A1/A2/A3/B axes in `metabench/hch/metagame_spec.md`.

**The core design failure we're trying to avoid:** construction problems like TSP and job-shop have named heuristics (NN+2-opt, Johnson's rule, list scheduling) that frontier models have *memorized*. They mentally execute the recipe in one pass, hit ~0–5% gap, stop. No multi-turn, no metacog signal. We have run this empirically — see `voicetree-15-4/jobshop-local-spike-ceiling.md` and `voicetree-15-4/tsp-structural-limits-problem-class-decision.md` for the evidence.

**What we need instead:** an NP-hard problem class where
- there is no single named algorithm / recipe that generalizes across instances
- the right heuristic is **instance-specific** — the model must analyze *this specific instance* to decide what approach to use
- solutions genuinely improve with more search time ("progressively better with effort")
- exact offline gold is computable in minutes via a real solver
- procedurally-generated instances (defends against training contamination)
- mechanically verifiable answer
- NL-renderable surface
- scoreable partial credit even if suboptimal

Chess is the canonical example of "instance-specific heuristics" — every position requires fresh analysis, no recipe transfers. We want the single-player / optimization analogue.

# What to research
Survey the academic literature (and practitioner solver-design literature) for NP-hard / PSPACE-hard search problems where the **heuristic is instance-specific**. I've sketched a tiered taxonomy below as a starting point — validate it, extend it, correct it.

**Tier 1 — general recipe exists, too expensive to mentally run:** Sokoban (IDA*+PDB), N-puzzle, classic TSP / jobshop. Our current fallback.

**Tier 2 — instance-dependent heuristic choice within a known framework:**
- Generalized PDDL planning (FastDownward ships a heuristic portfolio)
- Max-SAT at phase transition (ratio ≈ 4.26)
- General CSP with varied constraint structure
- Multi-agent path finding (CBS / EECBS / ICTS — each wins on different instances)
- Integer programming (branching / cut-selection is instance-specific)
- Quantified Boolean formulas

**Tier 3 — no general framework; each instance demands fresh analysis:**
- General Game Playing (GGP)
- Program synthesis / ARC-AGI
- Theorem proving (tactic choice)
- Custom energy-landscape optimization (protein folding variants)

# Specific things to dig into
1. **Are there classes I'm missing?** E.g., is there a well-known literature on "algorithm selection" / "per-instance solver configuration" that names problem families where this is especially acute?
2. **For each candidate class: how strong is the evidence that no single heuristic dominates?** Cite specific papers, solver competition results (SAT Competition, IPC planning competition, CP competition), or portfolio-solver results.
3. **Which classes are NL-renderable without losing their computational hardness?** (A domain that only matters in formal notation is a weaker benchmark target.)
4. **Which classes have existing open-source optimal solvers we could use for offline gold?** Name them (FastDownward, Z3, OR-Tools, Kissat, CBS solvers, etc.).
5. **What's known about LLM performance on each class?** Any prior benchmark results — ARC-AGI, PlanBench, etc.
6. **Training contamination risk** — is this class well-represented in common training corpora (papers, code, tutorials)?
7. **Which classes most naturally support "progressively better with more search time"** — i.e., anytime solvers with clear quality curves?
8. **Novel/underused candidates:** Are there problem classes that aren't in the standard benchmark canon but fit our criteria well? Look for classes from operations research, combinatorial chemistry, logistics optimization, circuit design, etc.
