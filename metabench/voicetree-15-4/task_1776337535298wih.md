# Portfolio spike v1 — structured-output upgrade + rerun

# Context
Tara built `hch/portfolio_spike/` and ran it once; the run died at turn 1 (`turn1_died`) because the regex label-block parser rejected Gemini's native JSON output. See progress node `portfolio-spike-v1-local-harness-and-run-2026-04-16.md`. Tara applied a dual-parser band-aid post-run but didn't rerun (single-run constraint).

The right fix is NOT dual-parsing. It's **structured output** — use Gemini's `response_schema` to schema-constrain the model's emission, delete the regex parser entirely. Same for exec turns. This eliminates the entire class of format-mismatch failures and will be the Kaggle port's eventual pattern too.

# Goal
1. Upgrade `hch/portfolio_spike/protocol.py` to use Gemini structured output for BOTH plan and exec turns. Delete the regex-based label-block parser.
2. Rerun the portfolio spike ONCE on seed=1, `models/gemini-3-pro-preview`, 30-min total wall budget. Capture the execution data Tara's death prevented us from getting.
3. One progress node.

# What to change

## `gemini_client.py`
Add support for `response_schema` parameter on `GeminiChatSession.send()` or equivalent. Gemini's API accepts a `response_schema` alongside `response_mime_type="application/json"` — check the SDK version Tara installed. If the existing `genai` SDK is too old, update the requirements file.

## `protocol.py`
Define two Pydantic models (or TypedDicts + explicit JSON Schemas):

```python
class PlanTurnOutput:
    PLAN: list[PlanItem]         # each: {id, problem, desc, budget_s}
    NEXT_SUB_ID: int

class ExecTurnOutput:
    SUB_N_RESULT: str            # freeform text, model's working/artifact
    FORECAST: dict[str, ForecastCDF]   # keys like "P1","P2"; each: {p_within_5pct, p_within_10pct, p_within_20pct, p_within_50pct}
    REVISED_PLAN: list[RevisedPlanItem]  # each: {id, problem, desc, budget_s, status, realized_bucket?}
    NEXT_SUB_ID: int | str       # int or "stop_economic" | "stop_budget"
```

Pass the schema to the Gemini call. Remove all regex extraction. Parse the returned JSON directly (it's guaranteed valid against schema).

Keep everything else from Tara's implementation: monotonic-CDF normalization, pre-flight gap gate, economic scoring, per-problem Brier, single-run constraint.

## System prompt
Update to reflect that output is strict JSON (not label-blocks). Keep the explicit plan-as-state language ("you CAN and SHOULD add new subtasks..."). Keep the cost_per_second declaration.

## NOT changing
- Problem generators, gold paths, pre-flight gate thresholds, economic objective, value_caps, cost_per_second ($0.05)
- Turn-1 minimal-output design
- 30-min total wall budget
- Seed (1), model (Gemini 3 Pro)
- ONE run only
