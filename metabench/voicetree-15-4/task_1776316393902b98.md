# Protocol implementation checklist
1. **TSP instance**: 25 cities, integer coords in [0,100]², seeded. Compute nearest-neighbor baseline tour. Compute near-gold via 2-opt-to-convergence (no solver dep needed for spike).
2. **Problem statement** includes coords, `BASELINE_TOUR`, `BASELINE_LENGTH`, the output contract, and the arm-specific system instruction.
3. **Turn 1 (planning only)**: run inside a `ThreadPoolExecutor` with `result(timeout=PLAN_TURN_BUDGET_S)`. Parse `ATOMIC_PREDICTION`, `DECLARED_GAP`, `DECISION`, `NEXT_SUB`. If timeout or parse fail → record `turn1_died=True`, final answer = baseline.
4. **Turn N≥2 loop**: before each turn, send a timing-stats user message (prev turn wall-seconds, input/output tokens, cumulative wall-clock, remaining budget). Run the turn with `SUBTASK_BUDGET_S` timeout. Parse `BEST_GUESS`, `P_CORRECT`, `DECISION`, `NEXT_SUB`. Update `last_best_guess`.
5. **Termination**: `DECISION: stop` OR total wall >= `TOTAL_BUDGET_S` OR turn timeout with no prior BEST_GUESS improvement slot. Use `last_best_guess or baseline`.
6. **Scoring (post-hoc in analyze.py)**:
   - `gap_pct = 100 * (tour_length(final) - gold_length) / gold_length`
   - `brier = (declared_gap - gap_pct)²`
   - `$score = 1.0 * max(0, 100 - gap_pct) - 0.01 * total_wall_seconds` (tune constants; just needs to be Pareto-meaningful)
   - Print a table: model × arm → mean gap, mean wall, mean $score, mean Brier, protocol flags count.
