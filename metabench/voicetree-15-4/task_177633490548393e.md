# Economic objective

**Per-problem value:** `V_i = value_cap_i × max(0, min(1, (baseline_i - model_i) / (baseline_i - gold_i)))`  — fraction of available headroom captured, clipped [0, 1].

**Session score:** `Σ V_i - cost_per_second × total_wall_seconds`

**cost_per_second:** visible to the model, declared up front in the prompt. Tune so that using the full 30-min budget ≈ 40% of max portfolio value → real allocation pressure. (Max portfolio value = 50+60+20+100 = 230. 30 min = 1800s. So cost_per_second ≈ 230 × 0.4 / 1800 ≈ $0.051/s. Round to $0.05/s.)

**Stop is rational when** remaining_expected_value < cost_per_second × remaining_wall_s.

# Turn structure

## Turn 1 (minimal planning output)
**Wall budget for turn-1 reasoning:** 5 minutes (restores benchmark-style planning budget, addresses previous 'turn1_died' / timeout issues).

**Required turn-1 output (minimal):**
```
PLAN: [
  {id: 1, problem: "P1"|"P2"|"P3"|"P4", desc: "<approach — short, don't restate problem>", budget_s: <int>},
  ...
]
NEXT_SUB_ID: 1
```
No per-problem forecasts required in turn-1. Problem specs live in the system prompt — `desc` is just "nearest-neighbor + 2-opt on P3", not a restatement.

## Turn N (exec + plan revise)
**Required output after each subtask:**
```
SUB_<N>_RESULT: <work + candidate artifact for the problem it targeted>
FORECAST_P<id>: {p_within_5pct: <float>, p_within_10pct: <float>, p_within_20pct: <float>, p_within_50pct: <float>}  // thresholded CDF, monotone non-decreasing
REVISED_PLAN: [
  {id: 1, status: "done", realized_bucket: "within_10pct"|"within_20pct"|"within_50pct"|"miss"},
  {id: 2, problem: "P2", desc: "...", budget_s: <int>, status: "pending"},
  {id: 5, problem: "P1", desc: "NEW: ...learned from sub 1...", budget_s: <int>, status: "pending"}   // additions OK
]
NEXT_SUB_ID: <int> | stop_economic | stop_budget
```

**Explicit revisability:** plan is stateful. Subtasks can be marked done, revised (budget/desc), added (from learnings), or skipped. System prompt must explicitly tell the model: "you CAN and SHOULD add new subtasks to the plan as you learn more from earlier subtasks — don't stop just because your initial plan is done if more value is available."

## Forecasts (Rex's fix restored)

Thresholded distance-to-optimal CDF, not exact correctness:
- `p_within_5pct`, `p_within_10pct`, `p_within_20pct`, `p_within_50pct`
- Monotone non-decreasing — enforce in parser
- Emitted when the model first plans a subtask for a problem AND after each subtask that touched that problem
- Scored via per-threshold Brier (4 Briers per problem per emission)

# Metacog signals to log

- Turn-1 wall time (so the model sees in turn-N context how long even minimal decisions took)
- Declared allocation (per-problem budget share) vs realized return per problem
- Number of distinct subtasks executed (primary question: does plan-as-state unlock multi-subtask behavior?)
- Plan evolution: additions, revisions, status flips per turn
- Per-problem gap and per-problem thresholded Brier
- Stop reason: `stop_economic` (model called it), `stop_budget` (budget exhausted), or timeout
- Wall + token cost totals
- Abandonment: any problem with budgeted subtasks never executed?
